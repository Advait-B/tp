## Overview

FlashCLI is a command-line flashcard application that allows users to create decks, manage flashcards, and study them interactively. It is designed to support efficient, keyboard-based learning without requiring a graphical interface.

My primary contribution was implementing the core command architecture of the application and ensuring its correctness through comprehensive testing.

### Implementation of command classes

I implemented multiple command classes that form the core functionality of FlashCLI. These classes are responsible for handling user input, executing application logic, and interacting with other components such as the Parser and UI.

Examples of commands I implemented include:
- `AddCardCommand`
- `ClearDeckCommand`
- `CreateDeckCommand`
- `DeleteCardCommand`
- `DeleteDeckCommand`
- `ExitCommand`
- `HelpCommand`
- `ListCardsCommand`
- `ListDecksCommand`

I also ensured that these commands follow a consistent structure and integrate cleanly with the overall command-based architecture.

**Impact:**  
These command classes form the backbone of the application, enabling all user interactions with decks and flashcards through the CLI.

---

### Integration with Parser

I integrated the command classes with the `Parser` so that the parameters from the userinput extracted by the parser are correctly used in constructing the appropriate command objects. 

**Impact:**  
This ensures that user commands are reliably interpreted and executed, providing a smooth and predictable user experience.

---

### Command-related test cases

I wrote comprehensive JUnit tests for command classes to verify both functional correctness and user-facing behavior.

This included:
- Testing valid command execution
- Testing invalid inputs and error handling
- Verifying CLI output using output stream capture

**Impact:**  
These tests improve software reliability and help prevent regressions, especially for user-facing CLI output.

**Note:**  
JUnit testing of CLI output has only been implemented for classes that do not create any other identifiable change to any objects, namely `HelpCommand`, `ListCardsCommand` and `ListDecksCommand`.
